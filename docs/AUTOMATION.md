# Documentation Automation

_Generated by `python scripts/sync_docs.py`. Update the generator if this policy changes._

## Why docs were drifting

- The only existing validation checked versions and changelog entries; it did not verify that repository-facing docs were regenerated.
- CI workflows lived under `cicd/aether-cicd/.github/workflows/`, which GitHub Actions ignores because workflows must exist under the repository root `.github/workflows/`.
- As a result, documentation checks were not enforced for normal pushes and pull requests.

## Permanent prevention

1. `scripts/sync_docs.py` regenerates deterministic documentation artifacts from the live repository tree.
2. `.github/workflows/repo-health.yml` runs the root test suite, the ML test suite, syntax checks, `sync_docs.py`, and `validate_docs.py` on every push and pull request.
3. The workflow fails if generated documentation changes were not committed, preventing silent drift.
4. On pushes to `main`, the workflow auto-commits regenerated documentation so the branch stays self-healing.
5. GitHub branch protection must require the `Repo Health / validate` status check on `main`; otherwise failing checks can still be bypassed by direct pushes or unreviewed merges.
6. `.github/workflows/repo-health.yml` and `docs/AUTOMATION.md` are both generated from this script, so when they conflict during a merge you should resolve `scripts/sync_docs.py` first and regenerate the derived files instead of hand-merging them.

## Branch protection setup

To remove GitHub's `Your main branch isn't protected` warning and make the automation enforceable:

1. Open **Settings → Branches → Add branch protection rule** (or **Rules → Rulesets** on newer GitHub UI).
2. Target the `main` branch.
3. Enable **Require a pull request before merging**.
4. Enable **Require status checks to pass before merging** and select `Repo Health / validate`.
5. Enable **Require branches to be up to date before merging** so the required check runs on the exact commit being merged.
6. Enable **Block force pushes** and **Block deletions** for `main`.
7. Optional but recommended: enable **Require conversation resolution before merging** and restrict who can bypass the rule.

## Merge-conflict recovery

If a pull request reports conflicts in `.github/workflows/repo-health.yml` and `docs/AUTOMATION.md`:

1. Update your branch with the latest base branch changes.
2. Resolve conflicts in `scripts/sync_docs.py` first, because it is the source of truth for both generated files.
3. Run `python scripts/sync_docs.py` to regenerate `.github/workflows/repo-health.yml` and `docs/AUTOMATION.md`.
4. Run `python scripts/validate_docs.py`, `pytest`, and `pytest 'ML Models/aether-ml/tests' -q` before committing the merge resolution.

## Team workflow

- Before commit: run `python scripts/sync_docs.py && python scripts/validate_docs.py`, plus the relevant root/ML pytest suites for touched code.
- During review: confirm behavior-changing code also updated authored docs under `docs/` when applicable.
- After merge: rely on `repo-health.yml` to catch or auto-commit generated documentation drift.

# 00 — Vision & Core Thesis

## The thesis

AETHER's deepest moat is not "better prediction models." It is that AETHER can become the **absence-aware, graph-native, agent-economic intelligence system** for the hybrid Web2/Web3 + agent economy.

Competitors model **presence**: what the user did, what score the model emits, what path the journey took. Very few model **absence** well — and fewer still operationalize it. AETHER already has the substrate (Expectation Engine, Behavioral Continuity, graph layers, lake-driven mutations) to build this.

The v-Next vision: AETHER becomes the only system that answers, with cryptographic provenance:

1. **What happened?** (existing — event + entity + graph)
2. **What should have happened but didn't?** (P2 Counterfactual Runtime)
3. **What almost happened?** (P10 Temporal Intelligence — Hawkes intensities)
4. **What caused this outcome across human + agent + wallet + payment?** (P1 Mission Graph)
5. **Which agent can we trust to act on our behalf, with what authority?** (P3 Agent Credit)
6. **Where is our graph thin, stale, or wrong?** (P4 Coverage Autopilot)
7. **Which subgraphs are coordinated/synthetic rather than organic?** (P5 Collusion Detection)
8. **What does the universal embedding of any entity look like?** (P6 Graph FM)
9. **Can we prove what we believed and when?** (P7 Provenance Substrate)
10. **What do we know that no single tenant could see alone?** (P8 Federation)
11. **When should the system abstain?** (P9 Safety Mesh)

## Glossary (terms introduced by v-Next)

| Term | Meaning |
|---|---|
| **Mission** | First-class record binding a human goal → delegated tasks → sub-agents → services → payments → protocol actions → outcome. Unit of causal attribution. |
| **Gap** | A specific instance of "expected action/edge/transition that did not occur." Ranked and addressable. |
| **Balance Sheet** (agent) | Persistent operating record: calibration, success rate, rework, spend efficiency, hire depth, downstream errors, authority utilization. |
| **Authority Band** | Scoped delegation level: `read-only`, `recommend-only`, `act-with-approval`, `bounded-spend`, `bounded-hire`, `bounded-onchain-action`. Computed from balance sheet. |
| **Witness Signature** | Cryptographic signature on ingested event, generated by SDK-held device-bound keypair. Proves event origin. |
| **Bitemporal** | Every vertex/edge property carries `(valid_time, transaction_time)`. Queries take `as_of` parameter. |
| **Conformal Prediction Set** | Calibrated prediction set with guaranteed coverage. Enables principled abstention. |
| **Coverage Debt** | Measurable deficit in graph completeness, freshness, evidence sufficiency, or confidence. |
| **AETHER-GPT** | Self-supervised foundation model pretrained on typed random walks over the unified graph. Single universal entity encoder. |
| **PSI** | Private Set Intersection — cryptographic primitive enabling cross-tenant identity matching without raw data disclosure. |
| **ZK Attestation** | Zero-knowledge proof of trust threshold (e.g. `trust ≥ 0.85`) verifiable on-chain without revealing underlying scores. |
| **Motif** | Graph pattern indicating coordinated/synthetic behavior (circular hire chain, payment loop, etc.). |
| **Shadow Lake** | Forked lake + in-memory graph for dry-run simulation of policy changes, agent behavior, mutation impacts. |
| **Hypergraph Edge** | N-ary relationship (DAO vote with N participants, multi-sig tx, x402 payment channel, agent crew). Modeled as `HyperEdge` vertex. |

## Guiding principles

1. **Additive only.** Every v-Next artifact is additive. No existing behavior changes semantics.
2. **Flag everything.** Every new path is gated; default off; opt-in per-tenant.
3. **Prove, don't assert.** If the system claims a fact, it can produce the evidence chain.
4. **Abstain when uncertain.** Conformal abstention is preferred to confident wrongness.
5. **Human-in-the-loop for irreversible actions.** No amount of model confidence bypasses Class 3/4/5 mutation approval.
6. **Compose, don't replace.** Mission Graph composes existing attribution + graph + events. Don't rebuild primitives.
7. **Security is a substrate.** Witness signatures, bitemporal, SBOM, conformal — non-negotiable Day 1.
8. **Documentation is code.** Every pillar has an API spec, a test plan, a model card, and a runbook before it ships.

## Non-goals for v-Next

- Replacing any existing ML model (all 9 stay; we finetune atop FM or add challengers)
- Replacing the existing graph schema (purely additive)
- Replacing the existing agent controller architecture (extended, not replaced)
- Multi-region deployment (out of scope)
- UI redesign (Shiki operator console gains new views, not a rewrite)
- Pricing model changes (commercial packaging is separate work)
